How this works. A GitHub Actions workflow fetches everyone's RS3 hiscores directly from Jagex every 2 hours and appends a snapshot to data/snapshots/ in this repo, tracked by a small data/manifest.json index. This page reads the manifest, then fetches only the two snapshot files it actually needs (the latest one, and whichever is closest to 7 days ago) — no CORS proxy in the loop, and no ever-growing file to download either. Snapshots older than 30 days are deleted automatically on each run. Weekly XP is the difference between those two snapshots, so figures firm up as more history accumulates.
